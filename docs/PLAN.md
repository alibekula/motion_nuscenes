# План Motion Prediction V1

Этот документ фиксирует решения по чистой V1-структуре без зависимости на старый архивный код.

## Базовые правила

- `dataloader.py` и `model.py` остаются отдельными файлами
- структура должна быть плоской и читаемой
- совместимость со старым пайплайном не добавляется без необходимости
- предпочтение отдаётся ясному коду, а не лишней абстракции

## Зафиксированные решения

- в основном V1-контракте нет `valid` и `missing` масок старого формата
- частичные траектории не являются основным сценарием для V1-артефактов
- окно по умолчанию: `history = 4`, `future = 12`
- `stride = 2` используется по умолчанию
- батчинг остаётся простым: padding до максимума внутри батча
- статическая карта строится один раз на `map_name`
- локальный map subset выбирается один раз на sample
- anchor bank строится offline
- обучение вынесено в отдельный `train.py`

## План по dataloader

1. Один раз построить scene index.
   Нужно сохранить:
   - упорядоченную timeline по каждой сцене
   - ego pose по кадрам
   - `map_name` для сцены
   - состояния агентов по кадрам

2. Один раз построить map store.
   На выходе:
   - глобальные polyline-объекты
   - статические map-объекты
   - достаточно геометрии для быстрых distance-checks

3. Собрать scene windows.
   Контракт:
   - `hist = 4`
   - `future = 12`
   - `stride = 2`

4. Жёстко фильтровать агентов.
   Оставлять только тех, кто:
   - существует в последнем history-кадре
   - присутствует во всех history-кадрах
   - присутствует во всех future-кадрах

5. Строить компактные тензоры.
   Основные поля:
   - `history_features`
   - `future_positions_ego`
   - `class_ids`
   - `agent_tokens`

6. Использовать чистый history feature vector размерности `18`.
   Состав:
   - `x, y`
   - `cos(yaw), sin(yaw)`
   - `vx, vy`
   - `yaw_rate`
   - `width, length`
   - `class_onehot[9]`

7. Локальную карту выбирать один раз на sample.
   Подход:
   - брать ego pose из последнего history-шага
   - выбирать top-k ближайших polyline/object записей
   - переводить только выбранный subset в ego-frame

8. Сохранять offline-артефакты.
   Train loader должен в основном:
   - читать `.pt`-артефакт
   - паддить батч
   - возвращать готовые тензоры

## План по модели

1. Держать основную модель в одном читаемом `model.py`.

2. Упростить history encoder.
   - без packed sequences
   - без старых temporal alignment path
   - GRU по фиксированной истории

3. Получать текущее состояние из последнего history-шага.

4. Сохранить базовую идею архитектуры:
   - history encoder
   - map encoder
   - scene fusion
   - anchor-based multimodal prediction

5. Оставить только действительно нужные mask-поля:
   - `agent_pad_mask`
   - `polyline_pad_mask`
   - `object_pad_mask`

6. Anchor bank хранить offline.
   Основные банки:
   - `A12`
   - `A6`

7. Лоссы считать в локальной системе координат относительно последнего history-состояния.

## Порядок работы

1. Довести `motion_v1/dataloader.py`
2. Получить корректный train/val артефакт
3. Подключить `motion_v1/model.py`
4. Запускать обучение через `train.py`
5. Только после этого удалять старые кодовые хвосты
